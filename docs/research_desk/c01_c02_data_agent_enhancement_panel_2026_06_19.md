## §2 분석 고도화를 위한 데이터 갭 (C-02 분석)

### 2.1 G1(변수 중요도) — 핵심 피처 D-015 미충족 항목
- **CPO_SBO_SPREAD** ❌: 대체재 임계값(>$175/MT) 핵심 피처인데 **CPO 선물(FCPOc1) 미수집**.
  → Bursa Malaysia CPO 선물 또는 MPOB 가격 추가 필요 (commodity_connector 확장).
- **GAIN PDF 정성 신호**: 작황·정책(수출세/관세)·재고 전망의 서술형 정보 → WASDE 발표 선행 신호.
  → `ingest_gain_oilseeds_pdf.py` + FinBERT/LDA 토픽 추출.

### 2.2 G2(가격 밴드) — 변동성·심리 레이어
- **COT(CFTC Commitment of Traders)**: 투기 순매수 포지션 — 인과체인에 이미 등장하나 데이터 없음.
- **VIX·달러인덱스(DXY)**: 위험선호·환율 채널.
- **에너지 연계**: 원유(WTI/Brent)·바이오디젤/SAF 수요 정책(RVO, EU RED III) — 식용유↔연료 수요 전환.

### 2.3 G3(레짐) — 시나리오 인코딩
- 위 거시 공변량 정렬 + ENSO 위상 더미 + 지정학 이벤트 프록시(이미 구현, 실시간 워크플로우 담당).

### 2.4 데이터 품질·정렬 (C-02 권고)
- 빈도 불일치(일별 CBOT vs 월별 WASDE vs 연별 PS&D) → G1 LASSO `dropna` 교집합 축소.
  → 월말 리샘플 + 적절한 ffill 한도 정책을 `_pivot_for_correlation`에 명문화 권고.

## §3 에이전트 아키텍처 검토 (참고 문서 반영)

**현황**: Nexus의 C-01~C-08·P1-01~05는 **개념적 페르소나**(스킬·룰 파일)로, 실제 Claude Code
서브에이전트가 아님. 참고 문서의 병렬화 수단(서브에이전트·에이전트뷰·에이전트팀·동적워크플로우)을
선택적으로 도입하면 반복 작업을 격리·병렬화할 수 있음.

| 도입 후보 | 용도 | 근거(참고 문서) |
|---|---|---|
| **서브에이전트: 커넥터 헬스체크** | 0건/키만료 진단을 메인 대화 오염 없이 수행·요약 반환 | "side task가 로그로 메인 대화를 채울 때" |
| **서브에이전트: 스키마 검증 게이트** | parquet ↔ `data/schemas/*.yaml` 정합성 검사(읽기전용, Write 미부여) | 도구 스코핑 — 최소 권한 |
| **동적 워크플로우/`/batch`** | GAIN PDF 900개 추출을 5~30 격리 워크트리로 분할 | "500-file migration / codebase-wide audit" |
| **WISC §5 Isolate 연계** | 10년 히스토리 감사 등 데이터 헤비 리서치 격리(이미 CLAUDE.md 명시) | 서브에이전트 결과 요약만 반환 |

**권고**: 신규 "페르소나" 추가보다 **기존 페르소나를 실제 서브에이전트(.claude/agents/*.md)로
구현**하고 도구를 최소 스코프로 제한. 특히 반복된 백필 실패 진단을 위한 **커넥터 헬스체크
서브에이전트**가 즉효성 높음.
