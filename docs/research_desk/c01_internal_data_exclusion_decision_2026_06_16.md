## 1. 사용자가 제시한 제외 근거

| 항목 | 내용 |
|---|---|
| **가용 데이터 한계** | 월별 입고량(procurement volume) + 기초/기말재고 수량만 존재. 일별 데이터 없음 |
| **OEM 생산 구조** | 대두유 원유는 사이로(silo)에서 생산에 필요한 만큼 수시 인출하며, 인출량이 매번 기록되지 않음 → 실소비량 추적 불가 |
| **시뮬레이션 제약 오염 위험** | 과거 조달 결정이 반영된 내부 데이터를 G2/G3 시뮬레이션 제약조건으로 사용 시, 모델이 시장 최적점이 아닌 과거 행동 패턴을 복제하는 방향으로 수렴 |
| **옵션 축소 위험** | 내부 재고 제약을 모델에 내재화하면 외부 시장 기회(spot 매수, 선물 헤지, 적기 대량 매수 등) 탐색 범위가 좁아짐 |
| **경쟁사 비교 불가** | 내부 행동 기반 모델로는 타사(해외 바이어, ABCD 트레이더)의 조달 패턴과 비교하거나 벤치마킹하기 어려움 |

## 2. C-01 × C-03 공동 기술 분석

### 2.1 내부 데이터 사용 시 발생하는 3가지 구조적 문제

**① 선택 편향 (Selection Bias)**  
현재 존재하는 내부 조달 기록은 과거 구매 결정의 결과물이다. 이 데이터를 학습에 사용하면 모델이 "과거에 얼마를 언제 샀는가"를 예측하는 모델이 되며, "시장 상황이 X일 때 최적 구매 타이밍은 언제인가"를 예측하는 모델이 되지 않는다. G2/G3의 목적은 후자이다.

**② 제약 오염 (Constraint Contamination)**  
시뮬레이션 파라미터로 내부 재고를 사용하면, 모델이 탐색하는 해공간(solution space)이 현재 창고 용량과 기존 공급업체 계약 조건에 의해 인위적으로 제한된다. 외부 시장에 존재하는 더 좋은 조달 창은 탐색되지 않는다.

**③ 월별 집계의 일별 모델 적용 불가**  
G2 가격 밴드 모델은 일별(daily) 주가/선물 가격을 입력으로 한다. 월별 입고량을 일별로 보간하면 허위 정밀도(false precision)를 만든다. 31일 중 실제 구매일은 1~3일이므로, 보간값은 30일 이상의 잘못된 데이터를 생성한다.

### 2.2 P1-04 공급망 관점

외부 CFR 가격 및 BDI 데이터만으로도 이론적 최적 조달 비용을 산출할 수 있다. 내부 실제 조달 가격과의 비교는 모델 출력 이후 **사후 검증** 단계에서 수행되어야 하며, 이것이 올바른 역할 분리이다.

### 2.3 P1-01 상품 시장 관점

글로벌 대두유 가격은 CBOT 선물, CPO 스프레드, FAS 수출 판매량이 결정한다. 특정 기업의 조달량(수백~수천 톤)은 전체 시장(연간 수백만 톤 수준)에 영향을 주지 못한다. 따라서 내부 조달 데이터를 외생 변수로 추가해도 G1 변수 중요도에 실질적 기여가 없다.
