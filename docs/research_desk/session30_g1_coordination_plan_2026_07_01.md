## 4. 데이터 공백 원인 진단 및 해결안 (구현 전 승인)

### 4.1 `GPR_NORMALIZED` 미수집
- **원인(확정)**: G1 임계값 로직이 `indicator_code == "GPR_NORMALIZED"`(임계 0.022) 조회. 그러나
  `gpr_connector`는 `GPR`·`GPR_QUALITATIVE`·`GPR_REALTIME`만 출력 → **정규화 파생 단계 부재(네이밍 불일치)**.
- **해결안**: 파이프라인에 GPR 정규화 산출(원본 → 표준화 → `GPR_NORMALIZED` 저장) 추가.
  임계 0.022는 정규화 스케일 기준이므로 정규화 산출이 정합. *(대안: 임계 로직을 실제 출력명으로 매핑)*

### 4.2 `BDI_ZSCORE` 미수집
- **원인(확정)**: `shipping_connector`는 원본 `BDI`만 출력. G1은 **90일 롤링 z-score(z>2σ)** 기대(dir="z").
  z-score **파생·저장 단계 부재** + 백필 BDI 히스토리가 90관측 미만이면 계산 불가.
- **해결안**: 피처 엔지니어링 단계에서 BDI 90일 롤링 z-score 파생·저장 + 백필 히스토리 ≥90 거래일 확보
  (무료 stooq/TE 커버리지 점검).

### 4.3 작황 / 수입통계 / 무역통계 / 수급전망 미수집
| 항목 | 원인 | 해결안 |
|---|---|---|
| 작황(crop) | production 커넥터 다수 실시간 전용 + 키 필요, BACKFILL 스킵 | NASA POWER(무료 백필) agromet + WASDE 생산·수율 파생을 작황 대리로 |
| 수입통계(customs) | 한국 관세청(data.go.kr) 실시간·키 의존, 백필 제한 | **UN Comtrade**(등록 키) 월별 수입 백필 + GATS 교차검증 |

## 6. ICE 거래량 데이터 활용안 (C-03 · P1 협의)

- **정의**: ICE 시장 월별 Futures/Options **거래량(volume)** — 가격이 아닌 **유동성·참여도** 지표.
- **가공(C-03)**: 20개 xlsx → 연도·시장·기초자산(Financial/Agri/Energy, EU=Oil/Energy)별 롱포맷 파싱 →
  `data/processed/ice_monthly_volumes.parquet` + 스키마 YAML. **농산물(대두 복합) 거래량**을 SBO 관련 신호로 선별.
- **피처 후보(P1-01·P1-04)**:
  - `ICE_AGRI_VOL_YOY`: 농산물 거래량 전년동월비 → 헤지·투기 참여 급증 = 변동성 선행.
  - `ICE_OIL_ENERGY_VOL_EU`: 유럽 Oil/Energy 거래량 → 바이오디젤·에너지 채널 활동도(P1-05).
  - 거래량 급증 월과 CBOT BO=F 변동성 상관 검증(G1 보조, G2 변동성 레이어 입력).
